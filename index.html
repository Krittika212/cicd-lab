CI/CD Pipeline

CI/CD stands for Continuous Integration and Continuous Deployment/Delivery. It is a method to frequently deliver apps to customers by introducing automation into the stages of app development.

Key Concepts

  • Continuous Integration (CI): Developers regularly merge their code changes into a central repository. Each merge triggers an automated build and test process.
  • Continuous Delivery (CD): Code changes are automatically prepared for a release to production. Deployment is manual but easy.
  • Continuous Deployment: Every change that passes automated tests is deployed automatically to production.

Popular Tools

  • GitHub Actions
  • GitLab CI/CD
  • Jenkins
  • CircleCI
  • Travis CI
  • Bitbucket Pipelines

Typical CI/CD Pipeline Stages

  1. Code: Developers write code and push to version control (e.g., Git).
  2. Build: The application is compiled or assembled.
  3. Test: Automated tests run to validate the code.
  4. Release: The build is packaged and prepared for deployment.
  5. Deploy: The application is deployed to staging or production environments.
  6. Monitor: Systems are monitored for issues and performance.
